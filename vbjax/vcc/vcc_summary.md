# Virtual Cohort Coordinator (VCC) - CrossCoder Pipeline Summary

This document summarizes the core components and design choices of the `CrossCoder` pipeline for amortized inference of whole-brain connectomes, based on the `vcc_modules.py` codebase.

## 1. Architecture & Weight Initialization
The `CrossCoder` is designed to map multiple "views" (e.g., different brain parcellations or imaging modalities) into a single, universal latent space. 
* **Model Type:** The core architecture consists of strictly single-layer linear networks (matrix multiplications) without hidden layers or non-linear activation functions.
* **Deterministic Mode:** Functions as a standard linear autoencoder. It initializes a single encoder weight matrix mapping input edges to a latent vector, and a decoder matrix mapping back.
* **Variational Mode (VAE):** Instead of a single point, it maps inputs to a probability distribution. The encoder splits into two heads:
  * **Mean Head ($\mu$):** Predicts the center of the latent distribution.
  * **Log-Variance Head ($\log(\sigma^2)$):** Predicts the log-variance. Log-variance is used because neural networks output unconstrained real numbers, but mathematical variance must be strictly positive (achieved later by taking the exponential). It's initialized with a large negative bias ($-10.0$) so the model starts with near-zero uncertainty and only increases variance where necessary.

## 2. Data Preparation (`add_view`)
Connectomes have wildly different scales and bounds depending on the modality. The `add_view` method normalizes the data before it enters the network:
* **`center`**: Subtracts the mean so every edge averages to exactly 0.
* **`zscore`**: Centers the data and divides by the standard deviation. A tiny epsilon (`1e-9`) is added to prevent division-by-zero errors for edges that are identically zero across all subjects.
* **`logit`**: Specifically useful for strictly bounded structural connectomes (e.g., Fractional Anisotropy bounded between 0 and 1). It squeezes the data slightly away from absolute 0 and 1, applies the logit function (inverse sigmoid) to stretch the values across the entire real number line $(-\infty, \infty)$, and then z-scores them. This allows the linear unconstrained network to predict freely without breaking mathematical bounds.
* **`nonneg` flag**: If True, any negative values produced by the linear decoder are instantly clipped to `0.0` to enforce biological reality (no negative structural tracts).

## 3. Loss Functions & Cross-Prediction
The "Cross" in CrossCoder comes from the nested loop inside the loss functions. Instead of just reconstructing view A to view A, the network encodes view $i$ and uses that single latent representation to reconstruct *every* view $j$. This forces the latent space to capture core biological information universal to all parcellations.
* **Reconstruction Loss (MSE):** The network minimizes Mean Squared Error, which mathematically equates to maximizing the Gaussian log-likelihood of the connectome data.
* **Variational Regularization (KL Divergence):** The VAE calculates a KL divergence penalty to force the learned distributions to resemble a standard normal distribution $\mathcal{N}(0, 1)$. This prevents the network from memorizing the data by shrinking the variance to zero.
* **The Reparameterization Trick:** To allow gradients to flow past the random sampling step, the network samples independent noise $\epsilon \sim \mathcal{N}(0, 1)$, then scales it by the predicted standard deviation and shifts it by the predicted mean ($z = \mu + \sigma \cdot \epsilon$).
* **Beta Annealing:** The KL divergence penalty is scaled by a parameter $eta$. Because connectome MSE sums over thousands of edges, a high $eta$ would overpower the reconstruction. $eta$ starts at 0 (allowing the network to learn to reconstruct first) and is slowly annealed to a tiny value (e.g., $10^{-5}$) to gently organize the latent space without losing detail.

## 4. Training Execution (JAX Specifics)
* **`jax.lax.scan`:** Instead of a standard Python `for` loop, the VAE training loop uses JAX's `scan` primitive. This compiles the entire iteration block into highly optimized XLA machine code that runs natively on the GPU without returning to Python between steps.
* **Gradient Clipping:** A `jax.tree.map` applies a `jnp.clip` to all gradients between -5.0 and 5.0 before updating the optimizer. This prevents exploding gradients, especially crucial when using the extreme values generated by the `logit` transform.

## 5. Post-Training Evaluation
* **`calc_mvn` (Total Cohort Variance):** In variational mode, modeling the population distribution requires the Law of Total Variance. The function calculates the covariance of the subject means and adds the average individual uncertainty ($\sigma^2$) to it. This produces a wider, better-calibrated prior distribution for downstream statistical generation.
* **`calc_confusion_rate` (Connectome Fingerprinting):** Evaluates if the network preserves individual subject identities. It encodes a subject's view A, decodes it into view B, and calculates the Euclidean distance to every empirical connectome in the test set. If the reconstructed subject #5 is mathematically closest to the *actual* subject #5, it counts as a match. The Confusion Rate is $1.0 - 	ext{accuracy}$.
